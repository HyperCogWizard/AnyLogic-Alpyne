alpyne.data.model_data

class alpyne.data.model_data.ModelData(name, type_, value, units=None)[source]

Represents a single data element with a name, type, value, and (optional) units.

This class is what each of the collection types (Inputs, Outputs, Configuration, Observation, Action) are composed of.

static from_json(data: Dict[str, Any]) → alpyne.data.model_data.ModelData[source]

Expands the values in a parsed JSON entry (a dictionary).

Parameters

data – an entry in the list of objects provided by the server (e.g., in the template)

Returns

an instance of this object, based on the data provided

static from_pair(name: str, value: Any) → alpyne.data.model_data.ModelData[source]

Construct an instance based on the provided information.

Used as part of a workaround when it’s preferable to build the object from Python information, rather than via the template.

Note that since Python types are simpler than Java’s, the assigned type may not match up to what is set in the model. The alpyne app will accept certain compatible types.

Parameters

  • name – the element’s name

  • value – the element’s value

Returns

a ModelData object
